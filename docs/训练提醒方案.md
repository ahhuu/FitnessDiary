🛠️ App 训练提醒功能开发指令
指令背景： 用户需要在“我的”页面设置一个每日定时的训练提醒。该功能需要包含时间选择器、开关控制，并在到达指定时间时发出系统通知。

核心目标：

UI 增加： 在“我的”页面（个人设置区域）增加一个“每日训练提醒”的入口。

交互实现： 点击弹出 TimePickerDialog 让用户选择时间。

后台通知： 使用 AlarmManager 实现精准的每日定时唤醒，并发送通知。

📋 任务清单
1. UI 改造 (fragment_profile.xml)
在“数据周报”卡片下方，添加一个新的设置项。

组件： 使用 MaterialCardView（保持 8dp 间距）。

布局内容：

左侧：ic_notifications 图标 + 标题 "每日训练提醒"。

右侧：一个 MaterialSwitch (开关) 和一个 TextView (显示当前设定的时间，如 "18:30")。

逻辑： * 点击卡片或时间文字 -> 弹出时间选择器。

切换开关 -> 开启或关闭闹钟逻辑。

2. 闹钟与通知逻辑 (ReminderManager.java)
创建一个工具类来统一管理提醒：

setReminder(Context context, int hour, int minute):

计算距离下次提醒的毫秒数。

使用 AlarmManager.setRepeating 设置每日循环闹钟。

发送到 ReminderReceiver。

cancelReminder(Context context): 停止闹钟。

3. 广播接收器与通知构造 (ReminderReceiver.java)
继承 BroadcastReceiver。

onReceive:

创建 NotificationChannel (针对 Android 8.0+)。

使用 NotificationCompat.Builder 构造通知。

标题： "该训练啦！💪"

内容： "今天的训练目标还没完成，点击开始打卡吧。"

小图标： 使用你重绘后的“打卡”图标。

点击行为： 点击通知直接跳转到 App 的“打卡”页面。

4. 权限与声明 (AndroidManifest.xml)
添加权限：<uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM" /> (针对 Android 12+)。

添加权限：<uses-permission android:name="android.permission.POST_NOTIFICATIONS" /> (针对 Android 13+)。

注册 Receiver。

💻 核心代码生成要求
请给出以下关键代码：

fragment_profile.xml 布局片段：包含开关和时间显示的卡片代码。

ReminderManager.java：处理 AlarmManager 设置与取消的逻辑。

ReminderReceiver.java：处理接收闹钟并弹出 Notification 的逻辑。

时间选择交互：在 ProfileFragment.java 中调用 TimePickerDialog 并更新 UI 的代码。

💡 建议
默认时间： 建议默认开启时间设为 19:00，这是大多数人训练的黄金时间。

适配： 提醒用户在手机设置中开启“允许后台运行”或“自启动”，否则某些国产系统会杀死 AlarmManager。
