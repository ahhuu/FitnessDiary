# 2026-07-19

## 今日完成事项 (Completed Tasks)
1. **全新原生长冷启动入场动画 (New Native Splash Screen & Animation)**
   - 引入 `androidx.core:core-splashscreen` 库接管 Android 12+ 的冷启动过渡。
   - 脱离对 Lottie 等三方动画库的依赖，手写实现了极具高级感的原生 `SplashAnimationView`，包含几何中心圆点扩散与心率起伏路径绘制。
   - 搭建了 `SplashFragment` 作为动画承载页，利用 2 秒钟的视觉停留时间在后台并发预加载所有业务数据（饮食库、运动库、Room DB 用户信息等），并在数据就绪后平滑 Navigation 跳转，实现了**“零感知”数据加载**。

2. **入场动画过渡与防黑屏闪烁优化 (Splash Screen & Black Flash Fixes)**
   - 彻底解决了 Android 12+ 强制系统启动页（SystemUI 负责绘制）在部分深度定制 ROM（如 MIUI/HyperOS）上引发的冷启动黑屏、状态栏变黑等跨进程 Bug。
   - 针对 SystemUI 无法正确解析跨进程颜色引用（`@color/...`）以及排斥全透明图标的底层机制，移除了全透明矢量图标，改用两套硬编码十六进制背景色（浅色 `#F7F5F0`，深色 `#121212`）的 `ic_splash_bg.xml` 进行无缝衔接。
   - 为主应用主题 `Theme.FitnessDiary` 与 启动页主题 `Theme.App.Starting` 均配置了 `android:forceDarkAllowed="false"`，阻断了国产手机系统强行给未适配应用铺黑底或色彩反转的“霸道机制”。

2. **全局深浅色模式初始化时机重构 (App-wide Theme Initialization Refactor)**
   - 将读取 SharedPreferences 并在代码里动态设置 `AppCompatDelegate.setDefaultNightMode()` 的逻辑，从 `MainActivity.onCreate()` 提前到了顶层 `FitnessDiaryApplication.onCreate()` 中。
   - 确保了 APP 级进程一旦拉起，主题环境立即敲定，彻底杜绝了因 Activity 中途重建而导致的闪烁和状态栏白字/黑字错乱。

3. **数据大盘平滑展示 (Data Dashboard Smooth Entrance)**
   - 针对动画结束后进入 `CheckInFragment`（首页数据大盘）时，因 Room 数据库异步查询完成导致的“数据瞬间蹦出”的闪烁感进行了视觉打磨。
   - 为 `NestedScrollView` 添加了 `alpha(0f)` 初始隐藏，并在 `onViewCreated` 结尾设置 250ms 延时配合 250ms 淡入动画。使得首页所有卡片如同预先渲染好一般，跟随 Splash 丝滑浮现。


4. **数据备份方案强化 (Data Backup Strategy Enhancement)**
   - 修改了 `BackupManager.java` 的备份逻辑，不仅备份 SQLite 主库 (`database.db`)，还将 `SharedPreferences` 中的配置文件 (用于存储运动总时长、日历备注等核心配置数据) 也合并纳入 ZIP 备份包中。
   - 同步将 SQLite 的 `database.db-wal` 和 `database.db-shm` 文件一并备份，彻底杜绝了未提交事务导致的恢复数据丢失或状态回滚的隐患。

5. **“我的”页面 (Profile) 设置及 UI 架构升级 (Profile UI Architecture Overhaul)**
   - 对 `fragment_profile.xml` 进行了深度重构，将原先零散杂乱的 7 个细分设置区域大洗牌，重新合并压缩为高度内聚且命名整齐划一的 **“5 大 4 字区块”**：
     - `数据状态`: 收拢了身体数据、健身目标、健康分析。
     - `资产荣耀`: 收拢了勋章墙、动作库、常用食谱。
     - `设备工具`: 收拢了 1RM计算器、TDEE、三方健康服务连接等。
     - `通用设置`: 收拢了目标设置、单位设置、通知提醒、缓存管理。
     - `帮助关于`: 整合了引导回顾、意见反馈、推荐应用、检查更新以及关于。

   - 修正了“检查更新”原先放在通用设置等不合常规的设计，将所有系统说明性质入口全部归纳到了最后一个区块，大幅提升了列表视觉精简度和 Material 规范合理度。
   - UI 文本调整：将原本的“数据综合周报”更名为更有延展性且专业的“健康分析”；将“推荐给好友”简化为“推荐应用”。

6. **首页健康评分明细状态精确化 (Health Score UI Precision)**
   - 修改了 `CheckInFragment.java` 中的评分弹窗渲染逻辑：当本日没有任何训练计划安排 (`totalPlans == 0`) 时，训练栏位状态文本由强硬的“无训练”调整为更人性化、更符合常理的 **“休息日”**。仅当本日有计划却未完成打卡时，才显示“无训练”。

7. **全新肌肉疲劳度热力图与身体数据中心融合 (New Muscle Fatigue Heatmap & Body Data Integration)**
   - 全新开发了基于过去 48 小时真实训练数据的“肌肉疲劳度热力图”功能，并将其创造性地与“身体数据中心”的高级人体剪影无缝融合，实现了“一个模型，既能量维，又显疲劳”的极致体验。
   - **视觉对齐修复**：对生成的透明底剪影原图 (`bg_human_silhouette.png`) 进行了完美的上下左右透明边缘一刀切裁边处理（316x912），并重新测算绑定了各测量锚点（如脖围、腰围等）的 `Vertical Bias`，使得各指示线如激光制导般精准指向人体部位。
   - **异步数据渲染打通**：在拉取用户基本数据时，于子线程后台并发读取过去 48 小时的 `DailyLog` 和 `ExtraExerciseLog`，调用 `MuscleFatigueCalculator` 生成全身衰减疲劳表，然后在 UI 层渲染出极具科幻感的黄色/橙色/红色高斯模糊渐变光晕。
   - **极致 UI 抛光**：移除了原先测量数据两侧冗余的米白色磨砂背景块，改为全透明悬浮效果；并在底部加上了结构清晰的“肌肉疲劳度”四色图例，整体 UI 更趋向于现代沉浸式“全息投影”风格。

8. **挑战系统全面升级与重构 (Challenge System Comprehensive Overhaul - Phase 1 to 4)**

     - **“我的”选项卡置底**：调整 UI 布局，将「✏️ 我的」过滤芯片移至最右侧，使「全部、饮食、运动、作息、身心、我的」依次排开，逻辑更顺畅。
     - **专属橙白背景色**：为无明确分类的纯自定义挑战赋予了专属的“橙白（#FFF3E0）”底色，以区别于预设模板的粉/蓝/紫/绿。
     - **智能分类收纳**：新建时如果勾选了诸如“每日称重”、“高碳水”等具体校验类型，系统会自动将其归入对应的内置分类（饮食/运动/身心）；如果未勾选具体校验，则归入专享的“我的”分类。
   - **编辑、“另存为”与重置机制**：

     - **引入策略模式 (Strategy Pattern)**：将 `ChallengeManager` 内部巨大的 switch-case 校验代码解耦，抽离为 `ICheckInStrategy` 和各种独立的设计模式类（如 `FatLossStrategy`），严格符合开闭原则。
   - **全新双轨制与免罚机制 (Dual-Track Logic & Freeze)**：
     - **双重失败判定**：彻底抛弃死板的 21 天。引入“纪律限制 (Max Fails)”滑块和“数学绝对限制 (剩余时间加已完成不足目标)”双轨引擎。
     - **请假条机制 (Freeze)**：支持配置请假券，偶遇意外无法打卡时可点击“❄️ 请假”免于失败惩罚。
     - **统一提醒引擎**：将新建的带时间属性的挑战无缝对接到原有的 `ReminderManager` 和 `ReminderSchedule` 中，零冗余实现每日打卡推送。
   - **精准数据联动校验 (Data-Driven Validation)**：
     - **增肌饮食/减脂冲刺**：精准对接最新数据，直接读取 `user.getTargetProtein()`、`user.getTargetCarbs()` 和 `user.getTargetCalories()`，挑战目标全权跟随用户真实设定的营养基线。
     - **称重与用药挑战**：每日只需录入合法体重数据或勾选完清单内所有药物/补剂，对应打卡方块即可自动点亮（治好强迫症）。
     - 支持在新建挑战时自由混搭“称重、增肌饮食、规律服药”等快捷芯片 (Chips)。
   - **极致的打卡界面视觉与动效抛光**：

     - **顶部卡片“拔高”质感**：去边框、去半透明，换上带阴影悬浮超大圆角纯净卡片，增大 Padding；打卡网格则被专属圆角背景框独立收纳，如同精美的日历板。
     - **列表细节打磨**：加大底部小卡片圆角；Emoji 底座增加微阴影（悬浮徽章感）；副标题文字透明度和行距调优；“失败 0/X 次”采用柔和浅红背景 (#FFEBEB) 加深红文字 (#C62828)，对比度极佳不再突兀。
     - **Lottie 动效全覆盖**：引入 Confetti（五彩纸屑）标准动画库，面板叠加透明层，在成功打卡（1.5s 局部动效）及坚持通关最终弹窗时播放饱满震撼的庆祝撒花特效。并修复了首页卡片因刷新冲突导致的动画消失问题。
     - **荣誉墙与情绪联动**：打通情绪模块（可创建“情绪记录挑战”），所有通关挑战永久陈列于“我的荣誉墙”上，并新增 🔥“连击 X 天”火焰文字。

## 待办与已知系统级限制 (TODO & Known Limitations)
- **冷启动动画与系统深色模式的根本冲突**：在 Android 12+ 中，若手机系统全局为深色模式，但 APP 内强制为浅色模式。由于冷启动页的绘制发生在 APP 代码运行前，系统必然强制画出深色启动页，随后应用强转浅色引发无法避免的黑闪。
- **系统层快照缓存 (System UI Snapshot Caching)**：调试 SplashScreen 期间，系统的启动页快照具有极强的缓存滞留特性，通常需要彻底卸载重装或重启手机才能观察到颜色主题修改的最终效果。
